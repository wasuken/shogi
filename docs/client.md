# AIクライアント (client)

## 概要

`client`ディレクトリには、将棋の対局を行うためのAIクライアントが含まれています。各クライアントは異なる思考アルゴリズムを実装（または実装予定）しており、`battle-runner`から呼び出されて`server`と通信し、対局を行います。

現在、以下の3つのクライアントが存在します。

1.  **Minimax (`minimax`)**
2.  **MCTS (Monte Carlo Tree Search) (`mcts`)**
3.  **AlphaZero (`alphazero`)**

すべてのクライアントは`shogi.js`ライブラリを使用して将棋の盤面管理や合法手の生成を行っています。

## 各クライアントの詳細

### 1. Minimax (`client/minimax`)

- **アルゴリズム:** Minimax法（α-β枝刈り法）
- **実装:**
  - 現在、プロジェクト内で最も基本的ながらも完全な思考ロジックを持つクライアントです。
  - `src/index.ts`内の`minimax`関数で探索を行っています。探索の深さ（`depth`）は現在`2`に設定されています。
  - `evaluate`関数で盤面の評価を行います。現在の実装では、持ち駒の価値を単純に合計する非常にシンプルな評価関数となっています。
  - `findBestMove`関数は、探索結果から最善手と、その手の評価値を返します。

### 2. MCTS (`client/mcts`)

- **アルゴリズム:** モンテカルロ木探索 (MCTS)
- **実装:**
  - **現状はプレースホルダーです。**
  - `src/index.ts`の`findBestMove`関数は、MCTSのロジックを実装する代わりに、すべての合法手の中からランダムに1つを選択して返すようになっています。
  - 今後、本格的なMCTSアルゴリズムが実装される予定です。

### 3. AlphaZero (`client/alphazero`)

- **アルゴリズム:** AlphaZero
- **実装:**
  - **現状はプレースホルダーです。**
  - `mcts`クライアントと同様に、`findBestMove`関数はランダムに手を選択するだけです。
  - AlphaZeroはニューラルネットワークを用いた自己対局による強化学習を必要とするため、実装は他のアルゴリズムに比べて複雑になります。

## 共通のインターフェース

各クライアントは、`battle-runner`から利用されるために、以下の共通の関数シグネチャを持つ`findBestMove`をエクスポートしています。

```typescript
// shogi.jsのインスタンスを受け取り、最善手（IMoveオブジェクト）を返す
export function findBestMove(shogi: Shogi): IMove | null;
```
※ `minimax`は評価値も返しますが、基本的なインターフェースは共通です。

## ビルド方法

各クライアントは独立したTypeScriptプロジェクトです。`Makefile`のコマンドを使って、すべてのクライアントを一度にビルドできます。

```bash
# すべてのプロジェクト（クライアント含む）をビルド
make build

# 特定のクライアントのみビルドする場合
cd client/minimax
npm run build
```

ビルドすると、各クライアントのディレクトリ内に`dist`フォルダが作成され、コンパイルされたJavaScriptファイルが出力されます。
